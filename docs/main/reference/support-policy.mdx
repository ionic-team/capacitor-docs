## 社区维护

Capacitor 自诞生之日起就采用 100% 开源（MIT 许可）模式，并将始终保持这一原则。通过 Capacitor 的社区维护策略，开发者可以确信 Capacitor 是其跨平台应用的正确选择。Capacitor 团队定期发布新版本和错误修复，并非常欢迎社区提交拉取请求。

## Capacitor 维护与支持状态

考虑到时间和资源的限制以及支持不断发展的移动平台生态的需求，Capacitor 团队会适时发布新的主版本。但我们会竭尽所能确保版本迁移过程尽可能平稳。Capacitor 团队建议升级到最新版本，以获得最新功能、改进和稳定性更新。

各 Capacitor 版本的当前状态如下：

| 版本 |      状态       |    发布日期    |   维护结束时间   |  扩展支持结束时间  |
| :--: | :-------------: | :------------: | :--------------: | :---------------: |
|  V7  |   **活跃支持**  | 2025年1月20日  |       待定       |       待定        |
|  V6  |  扩展支持阶段   | 2024年4月15日  |   2025年7月20日  |   2026年1月20日   |
|  V5  |  已终止支持  |  2023年5月3日  |  2024年10月15日  |   2025年4月15日   |
|  V4  |  已终止支持  | 2022年7月27日  |  2023年11月3日   |   2024年5月3日    |
|  V3  |  已终止支持  | 2021年5月18日  |  2023年1月27日   |   2023年7月27日   |
|  V2  |  已终止支持  | 2020年4月3日   |  2021年11月18日  |   2022年5月18日   |

- **维护阶段**：仅修复关键错误和安全问题，不进行重大功能改进
- **扩展支持阶段**：不再向新客户提供支持

## 最低支持平台要求
